# Tutorials

Tutorials are complete learning tracks, not isolated snippets. Each one should leave you with a working state and a clear explanation of what you can safely change next.

:::tip Choose a track
Use the beginner track when you are learning the framework. Use the production track when you already have a workflow and want to harden it.
:::

## Beginner track

| Step | Tutorial | Goal |
| --- | --- | --- |
| 1 | Build your first agent | Create and invoke a minimal workflow. |
| 2 | Add a Python tool | Connect the agent to safe application actions. |
| 3 | Add state and memory | Persist useful context across invocations. |
| 4 | Build a two-agent handoff | Compose predictable multi-agent behavior. |
| 5 | Add checkpointing | Make threads resumable and easier to debug. |
| 6 | Expose the workflow through the API | Serve the workflow from a backend process. |
| 7 | Stream responses to the TypeScript client | Build the frontend integration path. |

## Production track

| Step | Tutorial | Goal |
| --- | --- | --- |
| 1 | Structure an AgentFlow project | Separate workflow code, tools, API, and configuration. |
| 2 | Add persistent storage | Choose storage and checkpointing defaults. |
| 3 | Add background tasks and callbacks | Make long-running work observable and maintainable. |
| 4 | Add evaluation | Test important agent trajectories before release. |
| 5 | Deploy the API | Run the serving layer with environment-specific settings. |
| 6 | Connect the frontend | Use the documented client contract from application code. |
