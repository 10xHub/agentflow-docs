# How-to guides

How-to guides answer focused questions for developers who already understand the basics.

Use these pages when you know the job and want the shortest practical route. Conceptual background belongs in [Concepts](../concepts/index.md); exact behavior belongs in [Reference](../reference/index.md).

:::tip Practical rule
A good how-to guide should start with the recommended default, show the minimal code or command, then explain the main tradeoff.
:::

## Common tasks

| Task | Guide |
| --- | --- |
| Add a tool | Create and register a Python tool, then expose it through an agent node. |
| Persist state | Choose and configure a checkpointer for threads and recoverable runs. |
| Stream output | Stream workflow responses through the API and into a client surface. |
| Handle uploads | Send images, documents, and other media to an agent workflow. |
| Connect a client | Call a running AgentFlow API with the TypeScript client. |
| Deploy | Package and deploy an AgentFlow service with environment-specific configuration. |

## Before you start

If you have not run a workflow yet, start with [Your First Agent](../get-started/first-agent.md). If something fails while following a guide, use [Troubleshooting](../troubleshooting/index.md) before changing architecture.
